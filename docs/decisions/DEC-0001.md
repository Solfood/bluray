# DEC-0001: Adopt engineering scaffold for AI-assisted development

- Date: 2026-04-24
- Status: accepted
- Owner: Solfood
- Marker: BLU-DX-0001
- Supersedes: N/A

## Context

The bluray codebase is developed iteratively with AI assistance. Without a shared continuity structure, each AI session starts cold — no memory of what was decided, what's in progress, or what the next step is. This leads to repeated context-setting, inconsistent commit messages, and decisions that aren't recorded anywhere.

## Intake

- Objective: Establish a lightweight governance scaffold so AI agents can orient quickly, work items are tracked, and decisions are preserved.
- In scope: CLAUDE.md, policies/project-policy.yaml, docs/work-index.md, docs/session-log.md, docs/decisions/ directory structure.
- Out of scope: Changes to application code, test suite modifications, CI/CD changes.
- Risk class: low
- Critical assets touched: none
- Dependencies: engineering-scaffold-template (Solfood/engineering-scaffold-template)
- Expected evidence: All scaffold files present, no placeholder strings remaining, CLAUDE.md startup protocol navigable.

## Options Considered

1. **Ad-hoc notes in README** — inconsistent, hard to machine-read, no session continuity.
2. **External project management tool** — adds tooling dependency, not co-located with code.
3. **Engineering scaffold template** (chosen) — co-located with code, AI-readable, low overhead.

## Decision

Adopt the engineering scaffold template. Place scaffold files alongside source code. Keep governance lightweight — no gates required for low-risk DX changes.

## Why

Minimal overhead for a solo/small team project. Session continuity for AI agents is the primary value. The marker system makes commits self-documenting.

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Scaffold files add noise to the repo | Scaffold files are in docs/ and policies/ — clearly separated from source |
| Overhead if team grows | Template can be extended; the gate system scales up as needed |

## Rollback Plan

Delete CLAUDE.md, policies/, docs/work-index.md, docs/session-log.md, docs/decisions/. No application behavior is affected.

## Evidence

Scaffold files created and verified: CLAUDE.md, policies/project-policy.yaml, docs/work-index.md, docs/session-log.md, docs/decisions/DEC-0001.md. No placeholder strings found.
